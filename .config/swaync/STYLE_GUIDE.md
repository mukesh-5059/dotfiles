# Eww Widget Style Guide

This document outlines the general style and design conventions used in this `eww` configuration. Use this guide to create new widgets that maintain a consistent look and feel with the existing setup.

## 1. Core Philosophy

The design is modern, minimalist, and dark-themed. Key characteristics include:

- **Dark Interface:** The primary background is a very dark charcoal (`#0E0E0E`), with slightly lighter shades for widget containers.
- **Rounded Corners:** A defining feature is the consistent use of rounded corners on all elements, from the main bar and side panels to individual buttons and progress bars. The standard border-radius is `12px` for main containers and `8px` or less for smaller elements.
- **Color Accents:** A muted, warm color palette (primarily browns, oranges, and grays) is used for accents, icons, and progress indicators. The main accent colors appear to be `$color1` (`#64413F`) and `$color2` (`#9E5241`).
- **Smooth Transitions:** Hover effects and revealers use smooth CSS transitions (`0.3s ease-in-out`) to create a fluid user experience.
- **Minimalism:** The design avoids clutter. Information is often hidden by default and revealed on hover, keeping the main bar clean.

## 2. Color Palette

The color scheme is defined in `colors.scss` and is intended to be generated by a tool like `pywal`. The colors are referenced by variable names.

- **Background:** `$background: #0E0E0E` (Very dark gray/charcoal)
- **Foreground/Text:** `$foreground: #F8F8F8` (Off-white)
- **Primary Accent:** `$color1`, `$color2` (e.g., Muted orange/browns)
- **Secondary Accent:** `$color4` (e.g., Desaturated brown/tan)
- **Grays:** A range of grays (`$color3`, `$color5` through `$color8`) are used for text, icons, and inactive states.

When creating new elements, container backgrounds should be a mix of `$background` and another color, for example: `background-color: mix($background, $foreground, 95%);` which creates a slightly lighter charcoal.

## 3. Typography

- **Font Family:** The primary font is `Iosevka`.
- **Font Weight:** `bold` is the standard font-weight for most text.
- **Sizing:**
    - **Bar Text:** `1.1rem`
    - **Widget Labels:** `0.7rem` - `0.8rem`
    - **Icons:** `1.1rem` - `1.2rem` for the bar, and larger (`20px`) for side-panel controls.

## 4. Layout & Components

### Containers & Modules

- **Main Containers:** The top bar (`main-bar`), side-bar, and pop-up windows (`calendar`, `music-widget`) share a common style:
    - `background-color`: A slightly lighter shade of `$background`.
    - `border-radius`: `12px`
    - `padding`: `4px` to `12px` depending on the widget.
- **Widget Modules:** Individual widgets within a container (like the resource monitor or device controls) are also enclosed in boxes with a `12px` border-radius and `8px` padding.

### Buttons & Icons

- **Icon Buttons:** Most interactive elements are simple buttons containing only an icon.
    - They have no background by default.
    - The icon color indicates the state (e.g., a bright accent for "on", a muted gray for "off").
    - A hover effect (subtle change in color) is common.
- **Control Buttons:** Larger icon buttons, like in the side-bar's device controls, have a background color: `background-color: mix($background, $color1, 90%);` with a `8px` border-radius.

### Bars & Sliders (`<scale>`)

- **Trough (background):**
    - `background-color`: A dark gray, e.g., `mix($background, $foreground, 80%)`.
    - `border-radius`: `5px`.
    - `min-height`: `4px` for horizontal bars, `min-width` for vertical.
- **Highlight (foreground/progress):**
    - `background-color`: An accent color (e.g., `$color5`, `$color6`).
    - `border-radius`: `5px`.
    - `min-height`: `8px` (slightly taller than the trough to create a distinct look).

### Progress Bars (`<circular-progress>` and vertical bars)

- **Circular:** Used for battery and memory in the bar. The track color is dark gray, and the progress color is an accent color.
- **Vertical:** Used in the resource monitor. They follow the same color scheme: dark gray trough, accent-colored progress, and rounded corners (`6px` to `8px`).

### Revealers & Tooltips

- **Revealers:** Used extensively in the top bar to hide labels (`wifi-label`, `bluetooth-label`) until the user hovers over the icon. Use a `slideright` or `slideleft` transition with a `300ms` duration.
- **Tooltips:** Have a dark background (`$background`), light text (`$foreground`), and an `8px` border-radius.

## 5. Rofi Menu Style

The Rofi menu seen in the screenshot perfectly complements the `eww` widgets. To replicate this style for other menus:

- **Background:** Use the same dark charcoal as the widget backgrounds.
- **Rounded Corners:** Apply a `12px` or similar border-radius.
- **Input Bar:** Style it consistently with other widget containers.
- **List Items:**
    - **Default:** Simple text on the dark background.
    - **Selected:** A light gray background (`$color0` or similar) with rounded corners (`8px`).

## 6. Notifications

Notification pop-ups (`.notif`) are styled as floating, self-contained alerts.

- **Container:**
    - `background-color`: A dark, tinted color, mixed from the primary accent and the main background (`mix($color1, $background, 20%)`).
    - `border-radius`: `12px`.
    - `padding`: `12px`.
- **Layout:**
    - A main `box` with `horizontal` orientation.
    - An icon/image is on the left, with `10px` of padding to its right.
    - A vertical `box` on the right contains the title and body text.
- **Typography:**
    - **Title (`.notif-summary`):** Uses a bright accent color (`$color2`).
    - **Body:** Uses the default foreground text color.
- **Icon:** A symbolic icon (like the bell) uses the main foreground color.
